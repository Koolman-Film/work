About this mockup

Excel upload from scanner = critical V1 feature (fallback when scanner direct API not ready). Multi-step flow: upload → async parse → review preview → commit.

Steps

  1. Step 1: Upload — download template (CSV format documented), drag-drop xlsx, ≤5MB
  2. Step 2: Processing — Inngest async parse · spinner + progress · "สามารถปิด tab ได้"
  3. Step 3: Preview — validation summary + filter tabs + table with row issues highlighted · "Quick action" buttons (skip dups, map unknown empIDs)
  4. Commit — bulk insert, audit log, toast

Key UI elements

  • Stepper — visual progress through steps · complete step = green check
  • Validation summary cards — 3 cards: ✅ valid count (green) / ⚠ issues (orange) / total
  • Filter tabs — segmented control: ทั้งหมด / Valid / Issues
  • Issues alert — orange-bordered card listing issues with quick-action buttons
  • Preview table — sticky header · row-level highlighting (red for errors, orange for warnings) · checkbox per row
  • Issue tags — color-coded: ⚠ EmpID ไม่พบ / ⚠ duplicate / ✓ OK

Async pattern

File uploads to Supabase Storage → Inngest event fired → parse async with ExcelJS → save preview to ImportJob table → 🔔 admin notif when ready.

Admin can close tab during processing — comes back to notification, opens preview.
