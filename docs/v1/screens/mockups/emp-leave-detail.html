Detail page pattern

This screen demonstrates the detail page pattern reusable across:

  • S-E5 Employee leave detail
  • S-E8 Employee cash advance detail (similar with receipt image)
  • S-N4 Admin employee detail (with edit + archive actions)

Pattern structure

  1. Status hero — colored top stripe (orange/green/red) + status icon + summary
  2. Detail card(s) — info rows (label/value) grouped by section
  3. Reason box — left-bordered quote-style highlight
  4. Timeline — chronological history of events with vertical line + dots
  5. Action button — context-specific (Cancel for pending, none for approved)

States covered

  • Pending — orange stripe · cancel action available
  • Approved — green stripe · approval timeline · admin note · linked attendance records
  • Rejected — red stripe · rejection reason from admin
  • Cancel modal — confirm before cancellation

Timeline component

Vertical timeline with colored dots:

  • Primary (default) — ongoing/current
  • Success — completed action (green check)
  • Danger — rejected/error (red X)
  • Muted — neutral past event
  • Future (dashed) — anticipated next step
