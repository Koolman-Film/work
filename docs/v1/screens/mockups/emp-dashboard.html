About this mockup

Employee app uses mobile-first layout with bottom navigation (5 tabs). Desktop view keeps same layout but centers within max-width 480px (employee rarely uses desktop).

Key UI elements

  • Hero card — gradient blue-to-blue-800, big tabular numerals, NetPay shows latest published month
  • 3 KPI minis — leave/absent/late count this month, neutral muted bg
  • Pending list — shows pending leave/advance, click → detail page
  • Quick actions — primary CTA "ส่งคำขอลา" + secondary "ขอเบิกเงิน"
  • Bottom nav — 5 tabs, active state has primary-600 color
  • Bell badge — unread notification count (red, fixed top-right of bell)

Behavior notes

  • Hero card click → navigate to /payslip/[latestMonth]
  • KPI mini click → /attendance with relevant filter
  • Pending row click → relevant detail page (/leave/[id] or /advance/[id])
  • "ดูทั้งหมด →" → /leave or /advance
  • Quick actions → form pages
  • Empty pending list → hide section entirely
  • No published slip → hero shows "ยังไม่มีสลิปประจำเดือน" (smaller text)
💡 Implementation: Single Server Action getDashboardData() returns all 5 sections in one fetch — avoid multiple roundtrips.
