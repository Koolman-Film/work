Multi-section form pattern

Big forms (12+ fields) use section nav + content layout instead of single long form. Sticky section nav lets user jump between sections. Same pattern reusable for other complex forms.

Layout

  • Section nav (left, 240px sticky): 4 sections numbered, complete state shows green check
  • Form content (right): all sections rendered in single scroll, anchored by ID
  • Sticky footer: required fields summary + Cancel/Save
  • Section dividers: visual breaks between sections

Field types showcase

  • Text input with required marker (red *)
  • Email input with format validation
  • Phone input with Thai mobile format
  • National ID with 13-digit + checksum (live validation)
  • Select dropdown for branch/department/role
  • Money input with ฿ prefix + tabular nums
  • Date input for start date
  • Radio group for employment status (Probation / Regular)
  • Textarea for addresses
  • Composite emergency contact (name + phone + relation)

States covered

  • Section 1-4 — navigate between sections via nav (or scroll)
  • Section nav indicators: active (blue), complete (green check), error (red ×)
  • Saved state — success toast + redirect
