About this mockup

Leave approval inbox = primary admin task. Shows list + drawer pattern — common across many admin screens.

Key UI elements

  • Filter bar: search input + filter chips (status, dept, type, date)
  • Data table: checkbox first column for bulk · sortable columns · row click → drawer
  • Bulk action bar: appears when ≥1 row selected · Approve/Reject buttons
  • D-N1 Drawer: slides from right · employee info → leave details → team conflict context → action buttons
  • Reject sub-form: appears below drawer footer when reject clicked, requires reason

States covered

  • Inbox list (default) — table with 4 pending requests
  • Bulk selected (3) — bulk action bar appears, rows highlighted
  • Drawer open — D-N1 with full leave detail · approve/reject buttons
  • Drawer — rejecting — reason textarea revealed
  • Approved toast — T-N3 success after action
  • Empty state — when filtered to 0 results

Drawer content sections

  1. Employee card — avatar + name + department
  2. Leave details — type, dates, duration, reason (highlighted box)
  3. Attachment — preview thumbnail if uploaded
  4. Team conflict — who else on leave same days (helps decision)
  5. History — past leave for this employee (last 6 months)
  6. Action footer — Approve (success) / Reject (outline danger)
