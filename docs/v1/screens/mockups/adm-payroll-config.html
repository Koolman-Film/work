About this mockup

S-N23 = key-value config editor for system-wide payroll formulas. Distinct from generic settings CRUD pattern because each row has documented "key" + helpful description + special input types per value.

Key UI elements

  • Settings sub-nav (left, 220px): 7 tabs for all settings pages — current page highlighted
  • Section grouping: 4 logical sections (Social Security, Attendance, Pay cycle, Insufficient funds) with icon + name + desc
  • Config rows: 2-column key (label + monospace key-name + help text) + value (typed input)
  • Input variations: percent (with %), money (฿ + suffix), select dropdown, freeform formula
  • Change indicator: orange border on edited input + "✏ เปลี่ยนจาก X → Y" pill below
  • Sticky save footer: shows pending change count + "ไม่ retroactive" warning + cancel/save actions

Why distinct pattern

Generic CRUD pattern (S-N18-22) uses table + add/edit drawer. Payroll config is different because:

  • Fixed set of keys (not user-creatable rows)
  • Each key has specific input type (not just text)
  • Complex documentation per key (formula, units, examples)
  • Sectional grouping for readability
  • Audit-critical (every change should be logged)
