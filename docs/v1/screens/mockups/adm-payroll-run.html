About this mockup

S-N13 is the most critical screen in V1 — payroll calculation + review + publish flow. 8 distinct states from initial to published.

Status flow

  • Initial (Pending) — month not yet calculated · Trigger button enabled
  • Trigger modal (M-N2) — confirms before kicking off Inngest fan-out
  • Running — Inngest fan-out fires 1 job per employee · realtime progress via Supabase Realtime
  • Draft — all slips generated · admin reviews, overrides, sees warnings
  • Override modal (M-N3) — edit single field with required Note · audit log entry
  • Publish modal (M-N4) — confirms publish action with consequences listed
  • Publishing emails — Inngest sends 1 email + PDF per employee · live progress
  • Published 🔒 — locked · all emails sent · employees notified · ready to PEAK export

Key UI elements

  • Status chip in title — visual indicator (Draft/Running/Published)
  • Summary card — gradient blue with grid of totals (income, deductions, net, by group)
  • Warnings card — orange-bordered, lists override-worthy issues by EmpID
  • Filter bar — quick "Warnings only" toggle
  • Data table — dense rows with all payroll fields tabular-aligned · ✏ icon on right per row
  • Warning rows — subtle orange tint background

Server actions used

  • triggerPayroll(month) — fires Inngest event
  • overrideField(slipId, field, value, note) — audit-logged
  • publishPayroll(month) — locks + fan-out emails
  • unlockSlip(slipId, reason) — for revision after publish
