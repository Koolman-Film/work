Two-step reset flow

หน้านี้ใช้ /reset-password path เดียว — แสดง step 1 หรือ 2 ตาม URL searchParam ?token=...

  • Step 1 (path: /reset-password) — ผู้ใช้กรอกเบอร์โทร → server ส่ง SMS OTP
  • Step 2 (path: /reset-password?token=xyz) — ผู้ใช้กรอก OTP → ตั้ง password ใหม่

วิธีลองใช้

  • คลิก step pill เพื่อสลับ Step 1 / Step 2
  • คลิก state pill ในแต่ละ step เพื่อดู state ต่างๆ
  • ลองพิมพ์ password จริงใน Step 2 — strength bar เปลี่ยน live
  • Toggle dark mode

Step 1 states

  • Default — empty form
  • Filled — phone entered, ready to submit
  • Loading — submitting
  • Success — info card "ส่ง OTP แล้ว" (X-A4 — same UI regardless of phone exists)

Step 2 states

  • Default — empty password fields
  • Valid — strong password + match → submit enabled
  • Loading
  • Success — auto sign-in + toast T-A5
💡 Privacy note (X-A4): Step 1 success UI shows same message regardless of whether the phone exists in DB — prevents user enumeration attacks.
