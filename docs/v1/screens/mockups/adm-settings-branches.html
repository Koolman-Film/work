About this mockup

S-N18 = generic CRUD pattern reusable for 5 settings pages: Branches, Departments, Accounting Groups, Leave Types, Holidays. Same pattern, different fields per type.

Pattern structure

  1. Settings sub-nav (left, 220px): 7 sub-pages — current highlighted
  2. Page header: title + subtitle (count summary) + [+ Add] primary button
  3. Search bar: filter list by name
  4. Data table: sortable rows · status badge · row-level actions (edit / delete)
  5. Add/Edit drawer (slide from right, 480px): form fields appropriate for entity
  6. Delete confirm modal (M-C1): with X-N7 protection (block if has dependents)

States covered

  • List view — 4 rows including 1 archived (faded)
  • Add drawer — empty form for new entry
  • Edit drawer — pre-filled form with delete link in footer
  • Delete blocked (X-N7) — modal warns "ลบไม่ได้ — มีพนักงานสังกัด" → suggest archive instead
  • Saved toast — success after save/delete

Per-entity field variations

  • Branches: name + address textarea + isActive switch
  • Departments: name + isActive (simple)
  • AccountingGroups: name + accountingCode (PEAK mapping) + description
  • LeaveTypes: name + defaultQuota + isPaid switch + requiresDoc switch + docAfterDays + resetPolicy select
  • Holidays: date + name + type (national/company) + workPayMultiplier + branch scope
