About this mockup

Audit log = compliance + investigation tool. Append-only (RLS prevents UPDATE/DELETE). Every Admin/Owner action logged with before/after JSON.

Key UI elements

  • Filter bar (6 filters): date range, actor, action, entity, search · grid layout responsive
  • Action tags: color-coded by action category (create=green, update=blue, override=orange, archive/delete=red, publish/approve=green)
  • Entity tags: monospace ID with descriptive context
  • Highlighted rows: override actions tinted orange (visual signal)
  • D-N3 Drawer: 540px wide · metadata grid + before/after JSON diff
  • Pagination: page numbers + ‹ › nav · "Showing X of Y" indicator

D-N3 Drawer structure

  1. Action metadata grid: When, Who, IP, User Agent, Entity Type, Entity ID, Action
  2. Diff tabs: Diff (default) / Before / After / Raw JSON
  3. Diff view: JSON with - lines (red) and + lines (green) highlighted
  4. Related links: "ดู Entity →" button to navigate to source page (if still exists)

Privacy / RLS

  • Audit log append-only — RLS blocks UPDATE/DELETE for everyone
  • Sensitive fields masked: password hash, OTP, full bank account number
  • Read access: Admin (Super) + Owner only
  • Export CSV available for compliance archive
