About this mockup

Profile = self-service screen. Employee can edit own contact info, notification prefs. Some fields are read-only (managed by Admin only).

Tab structure

  • ข้อมูล (default): personal info (editable per field) + work info (read-only — ติดต่อ Admin)
  • แจ้งเตือน: notification preferences per event × channel matrix
  • ความปลอดภัย (Phase 4): change password + connect LINE OA

Inline edit pattern

  • Default state: each field is read-only row with [แก้ไข] button
  • Click [แก้ไข] → row expands with input + Save/Cancel buttons
  • Click Save → atomic update (one field at a time) → row collapses to read mode + toast T-E3
  • Click Cancel → revert
  • Locked fields (salary, dept, job) show 🔒 icon + greyed (no edit button)

Notification preferences

  • Per event-type × channel matrix (in-app / Email / LINE Phase 4)
  • Some critical events have locked-on in-app (security — can't disable own approval/payment notifications)
  • Switch animations 200ms ease

LINE connect (Phase 4)

  • Green LINE-branded card
  • "เชื่อมต่อ LINE" → opens LIFF URL inside LINE app → returns userId → POST /api/line/link
  • After connected: show "เชื่อมแล้ว ✓ {LINE display name}" + [ยกเลิกการเชื่อม]
